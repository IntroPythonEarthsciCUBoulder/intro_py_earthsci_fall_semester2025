Course Description

Computer programming is virtually indispensable in modern geoscience. This course provides an introduction to scientific computer programming using the Python language. After completing this course, you will be able to:

  • Write a computer program in Python to solve a common scientific problem such as reading in and plotting data, analyzing a dataset by performing calculations on its values, or performing calculations using a mathematical model.
  • Explain the basic elements of the Python programming language, such as variables, data types, operations, and functions.
  • Use common scientific packages such as NumPy, SciPy, and Matplotlib.
  • Understand and modify scientific Python scripts written by others.
  • Draw on a portfolio of well-documented programming examples created during the semester for examples and reminders of how to write Python code for various types of application.

The material will be motivated using various geoscientific problems that involve programming.

Required Work and Grading

Work for the course includes:

  • Five "code paper" assignments (10% each)
  • Daily in-class notebook work (20%) (note: you can miss a few of these without any grade penalty)
  • A lightning presentation and accompanying slide about a Python package of your choice (5%)
  • A final project (presentation and report) (25%)

Code paper and in-class work will be done using Jupyter Notebooks. A Jupyter Notebook is an interactive document that you create and edit through a web browser. Notebooks can combine text, pictures, nicely formatted math, Python code, and plots. This combination makes them really useful for the work in this course. The first session will introduce you to Jupyter Notebooks. Papers should be turned in electronically, in the form of the notebook file, and are due by midnight on the given due date.

Use of AI tools

"Don't copy code that you don't understand."

Artificial intelligence (AI)-based Large Language Models (LLMs), like ChatGPT, Gemini, etc., have become popular tools for coding help (among other things). But to use them effectively and reliably, you really need to understand the fundamentals of programming; otherwise, you're likely to end up with code that doesn't work or is full of errors.

In this course, the general philosophy is that LLMs are here to stay, and it's worth knowing how to use them, but they should never get in the way of learning.

To that end, unless otherwise noted, use of LLMs in programming assignments will be permitted as long as you follow two principles: show that you understand your computer code, and include an explanation of what tool you used, and how you used it. (I'll elaborate on this more in class.)
